00002 -- This module looks at the data on the CMD line and waits to receive a response.
00003 -- It begins examining the data lines when the i_begin signal is asserted. It then
00004 -- waits for a first '0'. It then proceeds to store as many bits as are required by
00005 -- the response packet. Each message bit passes through the CRC7 circuit so that
00006 -- the CRC check sum can be verified at the end of transmission. The circuit then produces
00007 -- the o_data and o_CRC_passed outputs to indicate the message received and if the CRC
00008 -- check passed.
00009 --
00010 -- If for some reason the requested response does not arrive within 56 clock cycles
00011 -- then the circuit will produce a '1' on the o_timeout output. In such a case the
00012 -- o_data should be ignored.
00013 --
00014 -- In case of a response that is not 001, 010, 011 or 110, the circuit expects
00015 -- no response.
00016 --
00017 -- A signal o_done is asserted when the circuit has completed response retrieval. In
00018 -- a case when a response is not expected, just wait for the CD Card to process the
00019 -- command. This is done by waiting 8 (=PROCESSING_DELAY) clock cycles.
