00002 -- This is an FSM that allows access to the SD Card IP core via the Avalon Interconnect.
00003 --
00004 -- This module takes a range of addresses on the Avalon Interconnect. Specifically:
00005 -- - 0x00000000 to 0x000001ff
00006 --                              word addressable buffer space. The data to be written to the SD card as well
00007 --                              as data read from the SD card can be accessed here.
00008 --
00009 -- - 0x00000200 to 0x0000020f
00010 --                              128-bit containing the Card Identification Number. The meaning of each bit is described in the
00011 --                              SD Card Physical Layer Specification Document.
00012 --
00013 -- - 0x00000210 to 0x0000021f
00014 --                              128-bit register containing Card Specific Data. The meaning of each bit is described in the
00015 --                              SD Card Physical Layer Specification Document.
00016 --
00017 -- - 0x00000220 to 0x00000223
00018 --                              32-bit register containing Operating Conditions Register. The meaning of each bit is described
00019 --                              in the SD Card Physical Layer Specification Document.
00020 --
00021 -- - 0x00000224 to 0x00000227
00022 --                              32-bit register containing the Status Register. The meaning of each bit is described
00023 --                              in the SD Card Physical Layer Specification Document. However, if the card is not connected or the
00024 --                              status register could not be read from the SD card, this register will contain invalid data. In such
00025 --                              a case, wait for a card to be connected by checking the Auxiliary Status Register (UP Core Specific), and
00026 --                              a command 13 (SEND_STATUS) to update the contents of this register when possible. If a card is connected then
00027 --                              the Auxiliary Status Register can be polled until such a time that Status Register is valid, as the SD Card
00028 --                              interface circuit updates the status register approximately every 0.1 of a second, and after every command
00029 --                              is executed.
00030 --
00031 -- - 0x00000228 to 0x000000229
00032 --                              16-bit register containing the Relative Card Address. This address uniquely identifies a card
00033 --                              connected to the SD Card slot.
00034 --
00035 -- - 0x0000022C to 0x00000022F
00036 --                              32-bit register used to set the argument for a command to be sent to the SD Card.
00037 --
00038 -- - 0x00000230 to 0x000000231
00039 --                              16-bit register used to send a command to an SD card. Once written, the interface will issue the
00040 --                              specified command. The meaning of each bit in this register is as follows:
00041 --                              - 0-5 - command index. This is a command index as per SD Card Physical Layer specification document.
00042 --                              - 6       - use most recent RCA. If this bit is set, the command argument will be replaced with the contents of
00043 --                                              the Relative Card Address register, followed by 16 0s. For commands that require RCA to be sent as
00044 --                                              an argument, this bit should be set and users will not need to specify RCA themselves.
00045 --                              - 7-15 - currently unused bits. They will be ignored.
00046 --                              NOTE:   If a specified command is determined to be invalid, or the card is not connected to the SD Card socket,
00047 --                                              then the SD Card interface circuit will not issue the command.
00048 --
00049 -- - 0x00000234 to 0x00000235
00050 --                              16-bit register with Auxiliary Status Register. This is the Altera UP SD Card Interface status. The meaning of
00051 --                              the bits is as follows:
00052 --                              - 0 - last command valid - Set to '1' if the most recently user issued command was valid.
00053 --                              - 1 - card connected - Set to '1' if at present an SD card
00054 --                              - 2 - execution in progress -   Set to '1' if the command recently issued is currently being executed. If true,
00055 --                                                                                              then the current state of SD Card registers should be ignored.
00056 --                              - 3 - status register valid -   Set to '1' if the status register is valid.
00057 --                              - 4 - command timed out         -       Set to '1' if the last command timed out.
00058 --                              - 5 - crc failed                        -       Set to '1' if the last command failed a CRC check.
00059 --                              - 6-15 - unused.
00060 --
00061 -- - 0x00000238 to 0x0000023B
00062 --                              32-bit register containing the 32-bit R1 response message. Use it to test validity of the response. This register
00063 --                              will not store the response to SEND_STATUS command. Insteand, read the SD_status register at location 0x00000224.
00064 --
00065 -- Date: December 8, 2008
00066 -- NOTES/REVISIONS:
00067 -- December 17, 2008 - added R1 response register to the core. It is now available at 0x00000238.
